# NotificationService

Микросервис уведомлений на **.NET** с единым API и подключаемыми каналами доставки (Email / Telegram). Репозиторий содержит отдельные проекты для API, каналов уведомлений и общий Shared-модуль. 

> Цель сервиса: принимать запросы на отправку уведомлений, маршрутизировать их по каналу(ам) и обеспечивать единый контракт для интеграции с другими сервисами.

## Состав репозитория

- `NotificationService.API` — HTTP API (точка входа для других сервисов).
- `NotificationService.Email` — канал доставки Email.
- `NotificationService.Telegram` — канал доставки Telegram.
- `NotificationService.Shared` — общие контракты/DTO/утилиты, используемые всеми модулями.

## Возможности

- Единый API для отправки уведомлений.
- Разделение по каналам доставки (каждый канал — отдельный проект).
- Shared-библиотека для переиспользования моделей и контрактов.
- Подготовлено для контейнеризации (в репозитории используются Dockerfile).

### Требования
- .NET SDK
- Docker (опционально)

## Добавление нового канала

1. Создать новый проект, например `NotificationService.Sms`.
2. Подключить `NotificationService.Shared` (контракты/общие модели).
3. Реализовать обработчик/отправщик для канала.
4. Зарегистрировать канал в API (маршрутизация / DI / конфигурация).
